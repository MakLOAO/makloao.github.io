<c:out>显示表达式

<c:out>作用与脚本的<%=%>作用相似，但是<c:out>可以直接通过”.”操作符来访问属性。

语法格式如下
<c:out value="<string>" default="<string>" escapeXml="<true|false>"/>

属性 描述
value 要输出的内容，必要，可以设置为${class:function}输出函数的结果
default 输出的默认值，非必要
escapeXml 设置是否忽略XML特殊字符，默认为true，设置为false则输出HTML内容

<c:set>设置变量和对象属性

语法格式

属性 描述
value 要存储的值
target 要修改的属性所属的对象
property 要修改的属性
var 存储信息的变量
scope var属性的作用域

使用<c:set>设置属性变量var

1.没有体
<c:set var="userLevel" scope="session" value="Cowboy" />

2.有体
<c:set var="userLevel" scope="session" >Sheriff, Bartender, Cowgirl</c:set>

有体的情况下，会计算体并作为变量的值，如果值计算为null，变量会被删除

对bean和Map使用<c:set>

这一类<c:set>只能用来设置bean属性和Map值，不能用它来向列表或数组增加元素

1.没有体
<c:set target="${PetMap}" property="dogName" value="Clover">

如果target是一个bean，则property为属性，如果target是一个Map，则property为键。

2.有体
<c:set target="${person}" property="name" >${foo.name}</c:set>

体可以是一个String或表达式

<c:set>要点和技巧

  • <c:set>不能同时有var和target属性
  • scope是可选的，默认为page作用域
  • 如果value为null，var指定的属性将被删除
  • target必须放入一个能解析为实际对象的表达式，不能为String

<c:remove>删除元素

语法格式
<c:remove var="<string>" scope="<string>"

var为要删除的变量名，scope为变量的作用域
var属性必须是一个String直接量，不能是表达式

<c:import>包含内容

语法格式

属性 描述
url 待导入资源的URL，可以是相对路径或绝对路径
context 当使用相对路径访问外部context时，context指定了这个资源的名字
charEncoding 所引入的数据的字符编码集
var 用于存储所引入的文本的变量
scope var属性的作用域
varReader 可选的用于提供java.io.Reader对象的变量

三种包含内容方法的区别

1.include指令
    <%@ include file="Header.html" %>
    静态导入，在转换时将file指定的文件内容加入到当前页面

2.<jsp:include>标准动作
    动态，在请求时将page属性指定的内容添加到当前页面

3.<c:import>JSTL标记
    动态，与标准动作相似但更强大更灵活。

TLD配置文件

TLD描述了两个主要内容：定制标记和EL函数，上一章曾经用过只有一个<function>元素的TLD，这节还要用到<tag>元素，如下

解释TLD

TLD解释如下:
<tlib-version>必要，声明标记库版本
<short-name>必要，主要由工具使用
<uri>taglib指令使用的唯一名
<description>不解释
<name>必要，标记中使用的就是这个名（冒号右边）
<tag-class>必要，在JSP使用这个标记要调用什么
<body-content>必要，说明体内不可以有任何内容
<attribute>用来设置属性，rtexprvalue指定该属性是否可以是一个表达式，如果设置为false或者没有设置，则不能使用类似"${userName}"这样的表达式
