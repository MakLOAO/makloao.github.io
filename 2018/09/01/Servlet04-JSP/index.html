Servlet04-JSP

基本指令

page

使用page指令导入包
<%@ page import="foo.*" %>
<%@ page import="foo.*,java.util.*" %>

page指令的属性

taglib

taglib定义JSP可用的标记库
<%@ taglib tagdir="/WEB-INF/tags/cool" prefix="cool" %>

include

include定义在转换时增加到当前页面的文本和代码
<%@ include file="xxx.html" %>
使用此指令允许你建立可重用的块（如标准页面标题或导航栏），这些可重用的块能增加到各个页面上，而不用每个JSP都重复写这些代码

表达式

对于<% out.println(Counter.getCount()); %>
可以用<%= Counter.getCount() %>代替，不需要分号，表达式内的内容为out.print的参数

声明

<%! int count = 0; %>
声明该JSP所生成的servlet类的成员

覆盖jspInit()

只要在.jsp文件中实现了jspInit()方法，在JSP页面成为servlet的最开始容器就会调用这个方法，这个方法由servlet的init()方法调用，所以在运行jspInit()方法时，已经有一个ServletConfig和ServletContext可供servlet使用，可以在jspInit()中调用getServletConfig()和getServletContext()

同理jspDestroy()也可以覆盖

JSP的属性

除了Servlet拥有的请求，会话和上下文这三个属性，JSP还拥有页面作用域pageContext（隐式对象），常用于开发定制标记

使用PageContext得到属性

可以在PageContext引用得到任意作用域的属性

同理使用
<% pageContext.setAttribute("foo", "attr", PageContext.SESSION_SCOPE); %>
等同于
<% session.setAttribute("foo", "attr"); %>

使用
<%= pageContext.findAttribute("foo") %>
可以在不知道作用域的情况下查找一个属性

EL预热

EL(Expression Language)是无脚本JSP，可以允许开发人员在JSP中不适用Java也能达到同样的效果

为什么使用EL？

  1. 不应该要求WEB页面设计人员必须懂Java
  2. JSP中的Java代码很难修改和维护

开启无脚本JSP

在配置文件web.xml中

选择忽略EL
