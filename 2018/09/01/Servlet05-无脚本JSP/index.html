Servlet05-无脚本JSP

bean法则-使用bean的规范

  1. 必须有一个无参数的公共构造函数
  2. 必须按命名规范来命名公共的getter和setter
  3. setter的参数类型和getter的返回类型必须一致
  4. 性质名和类型是由setter和getter得出，而不得来自于类中的一个成员
  5. 结合JSP使用时，性质类型必须是String或其他基本类型

分析<jsp:useBean>和<jsp:getProperty>

<jsp:useBean>

用 <jsp:useBean> 声明和初始化一个bean属性，id为声明bean对象的标识符，对应于servlet的request.setAttribute("person", p)，class为声明的对象的类类型，scope标识这个bean对象的属性作用域

<jsp:useBean>还可以创建一个bean，如果它找不到一个名为”person”的属性对象，它就会建一个

<jsp:getProperty>

用 <jsp:getProperty>得到bean属性的性质值，name标识具体的bean对象，与<jsp:useBean>的id属性相匹配，property标识属性中的性质名(也就是与bean类中getter和setter对应的性质)

<jsp:setProperty>

<jsp:setProperty>是第三个（也是最后一个）bean标准动作，具体使用如下

<jsp:setProperty name="person" property="name" value="yourNameVal" />

<jsp:useBean>可以有体

如果把设置方法代码<jsp:setProperty>放在<jsp:useBean>的体中，就可以有条件地设置性质，只有创建新bean时才设置性质值

多态bean引用 <jsp:useBean>的type属性

可以用<jsp:useBean>的type属性实现引用类型不同于具体的对象类型，假设新对象的类型为Employee，引用类型是Person

<jsp:useBean id="person" type="foo.Person" class="foo.Employee" scope="page">

EL函数

使用EL函数有如下四步骤

  1. 编写一个有公共静态方法的Java类
  2. 编写一个标记库描述文件(.tld)
  3. 在JSP中放一个taglib指令
  4. 使用EL调用函数

编写一个有公共静态方法的Java类

编写一个标记库描述文件

IDEA中在web目录右键new -> xml configuration file -> JSP tag library descriptor 重命名为*.tld

在JSP中放一个taglib指令

<%@ taglib prefix="mine" uri="DiceFunctions" %>

使用EL调用函数

正文添加
${mine:rollIt()}

对TLD文件的解释

  • TLD中的uri对应JSP中taglib指令的uri属性
  • TLD中function里的name标签的值对应EL表达式中冒号右边使用的函数
  • TLD中function-class标签对应.class
  • TLD中function-signature标签的值对应.class里的公共静态方法及其返回值
  • taglib指令中的prefix属性的值对应EL表达式冒号左边的值

*.tld文件可以放在WEB-INF目录或者其某个子目录中

EL处理null

对于找不到表达式中指定的属性/性质/键，也会显示页面，在算术表达式中，EL把null值看做是0，在逻辑表达式中，EL把null看做是false

可复用的模板部件

对于网站上每个页面的页眉和页脚都一样的情况下，每个JSP反复编写同样的页眉页脚，会显得效率低下，这个时候可以使用include指令把相同的页面放在不同网页对应的位置上

<include>指令

用法：
<%@ include file="*.jsp" %>

<jsp:include>标准动作

用法
<jsp:include page="*.jsp" />

<include>与<jsp:include>的区别

include指令在转换时发生，<jsp:include>标准动作在运行时发生
include指令相当于复制代码粘贴，标准动作则根据页面属性创建一个RequestDispatcher，并应用include()方法。

注意！不需要把<html>和<body>标签放到可复用部件中
