SSM知识点补充

Spring MVC

Controller

基于注解开发(@Controller等4个)记得添加 <mvc:annotation-driven/>

在 springmvc.xml 配置文件中配置 view resolver 中的 prefix 和 suffix 分别加 Controller 中返回的字符串的前后，构成对应返回的页面的地址，如果 Controller 要重定向到其他请求，则使用 return "redirect:/xxx"

RequestMapping

前端表单请求的对象 <form action="login.html" method="post">，就是RequestMapping中的值加上在web.xml中配置的前端分发器 <url-pattern>*.html</url-pattern> 的后缀

常用的 url-pattern 有 / 表示前端分发器拦截所有请求， *.html 表示前端分发器拦截所有 .html 结尾的请求

RequestBody与RequestParam

@RequestBody 接收的参数来自请求体，一般用于处理非 Content-Type: application/x-www-form-urlencoded编码格式的数据，比如：application/json、application/xml等类型的数据，处理POST请求
@RequestParam接收的参数来自请求头

以POJO对象作为Controller的接收参数，前端传输来的参数（比如JSON）需要配置MessageConverter才能转换为POJO对象

整合

注意的点

mapper和接口类名一致，mapper的标签id与接口方法一致，然后配置好对应配置后，直接在service层可以使用接口对应的方法（Autowired装载到service层）

对Mapper多参数的处理

mapper多参数的时候，不可以在xml配置文件写属性 <mapper parameterType='xxx' >，解决方法有：

  • 用Map封装参数
  • 在对应接口的参数写 public String getName(@param("name") String name, @param("id") int id);

(@param指定参数后，如果参数是某个类，#{属性名}依然可以拿到属性值，但是在动态SQL中的表达式必须要用 类的参数名.属性）

多参数不能传实体对象，而是把实体对象的域一个个传，否则解析不了

实体属性与数据库字段名字不一致解决方案

  1. resultMap
  2. 字段别名与实体属性名相同
  3. Map集合封装（好处是多表查询可以用）
  4. 配置属性映射，如果数据库使用的是下划线，实体使用的是驼峰，可以使用MyBatis的下划线与驼峰命名映射，如下配置：
